# 刚果（金）冲突态势感知地图 · DRC Conflict Situational Awareness Atlas

面向中文读者的刚果民主共和国武装冲突态势感知交互地图。系统聚合 **ACLED**（武装冲突地点与事件数据项目）专家编码的冲突数据，覆盖刚果（金）26个省份自 2010 年以来的武装冲突事件，以地图、统计、时间轴和报告等形式呈现。

> 仅做公开信息的汇总与可视化呈现，不构成独立安全分析或预测。

## 在线访问

**GitHub Pages**: `https://snigellinofficial.github.io/-drc-conflict-atlas/africa_security_map.html`

也可以下载仓库，用浏览器打开 `africa_security_map.html` 即可查看。无需安装任何软件，无需 API 密钥，无需配置。地图底图从 CARTO 加载，需要网络连接。

## 数据来源

系统的冲突数据来自 **ACLED**（武装冲突地点与事件数据项目），总计 **35,810 条**真实冲突事件，覆盖 2010年7月至今：

| 数据源 | 说明 |
|--------|------|
| **ACLED** | 核心数据源。由专家团队编码，覆盖刚果（金）全境的战斗、爆炸、平民暴力和战略动态事件。经智能采样后展示核心事件。 |
| **GDELT** | 备用验证源。通过 NLP 从全球新闻中自动提取事件信息，用于交叉验证。 |
| **UCDP** | 学术验证源。乌普萨拉大学维护的全球武装冲突数据库，学界标准严格。 |

所有事件均标注来源与验证状态。数据通过 **GitHub Actions 每日自动更新**（UTC 00:07），无需手动操作。

## 功能导览

打开页面后你会看到一张覆盖刚果（金）26省的地图，左侧是地图区域，右侧是浏览面板。页面顶部有**中/英语言切换按钮**，除行为体名称等专有名词外，所有界面文字均可切换。

**地图交互**。省份以浅色填充显示，有冲突事件的省份会呈现较深的红色。点击任意省份会弹出该省的历史冲突概况（含行为体按频率排序）。散点标记代表单个冲突事件，颜色表示类型，大小表示严重程度。点击标记可查看事件详情。地图右下角有图例，左下角有比例尺和时间范围选择器。

**浏览面板**包含七个模块：

1. **统计概览** — 六张卡片总结当前筛选条件下的关键数字。
2. **筛选器** — 按冲突类型、严重等级、参与行为体和省份过滤事件。支持全选/反选，搜索框可快速定位特定行为体。点击行为体旁的 i 图标可查看其组织档案。
3. **事件列表** — 列表展示所有匹配事件，支持按日期、死亡人数或严重等级排序，也可按年份或省份分组。
4. **时间轴** — 按时间倒序展示事件，适合快速浏览最近动态。
5. **更新日志** — 记录每次数据更新的时间和新增条数。
6. **月度报告** — 点击自动生成过去 31 天的结构化安全态势报告。
7. **快速报告** — 输入关键词如"M23"或"Ituri"，系统会自动匹配并生成简式报告（中/英双语）。

**左侧时间选择器**提供七个快捷按钮：1个月、3个月、6个月、1年、3年、5年、全部，也可手动输入起止日期。省份高亮和标记会随时间段自动变化。

**弹窗导航**。右侧次级面板支持多级返回（左上角返回按钮逐级回退，右上角关闭按钮一次性关闭所有层级）。

**缩放行为**。地图缩放至 9 级显示省份中英文名称，10 级显示城市名称。

## 严重等级说明

| 等级 | 中文 | English | 标准 |
|------|------|---------|------|
| Critical | 严重 | Critical | 死亡 ≥ 50 人，或涉及大屠杀 |
| High | 高 | High | 死亡 20–49 人，或重大战役 |
| Medium | 中等 | Medium | 死亡 5–19 人，或武装冲突 |
| Low | 低 | Low | 死亡 < 5 人，或战略动态 |

## 数据更新

项目通过 **GitHub Actions** 每日自动从 ACLED API 拉取最新数据，合并后提交至仓库。用户只需 `git pull` 即可获取最新数据，无需手动运行任何爬虫。GitHub Pages 在每次 push 后自动重新部署。

想了解数据更新详情，查看 `.github/workflows/update_data.yml` 和 `scripts/ci_update.py`。

## 技术说明

- **前端**：Leaflet 地图 + MarkerCluster 聚合 + CARTO 浅色底图（羊皮纸美学）
- **省份边界**：geoBoundaries GeoJSON 数据
- **架构**：前后端分离，数据层与展示层严格分离。所有数据通过 `<script>` 标签加载，不依赖服务器端处理，直接在浏览器中运行
- **i18n**：自定义轻量国际化引擎，无外部依赖，通过 URL 参数 `?lang=en` 持久化语言偏好
- **样式**：Noto Serif SC（中文）+ Times New Roman（英文），殖民时期羊皮纸配色

## 注意事项

- 本项目仅用于学术研究和态势感知。所有数据和分析均来自公开来源。
- 死亡人数为各来源报告的估计值，实际数字可能有所差异。
- 冲突事件的分类和严重等级判定参照 ACLED 标准，不代表任何政治立场。
- 数据主要来自 ACLED，辅以 GDELT 和 UCDP 进行交叉验证。
